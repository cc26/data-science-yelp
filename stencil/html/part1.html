Part 1: Search Engine

Description & Overview

Note: it's advised to run query on a computer with 8G of memory. The pre-processing time should be slightly above 1 min.

This project builds a simple search engine that is able to answer search queries on yelp business reviews.

The business data and review data is provided by Yelp as part of its academic dataset. We preprocessed the reviews, and made a program that can make inqureys for specific word(s), phrases, and find businesses sorted by relavance.

Steps

Create Inverted Index

Structure

The structure of the inverted index:
{word -> {business_id -> {review_id -> [positions]}}} in json form

First, we build an inverted index of the business data. The data we need to consider is the business data and the review data. The inverted index maps each stemmed words to a list of businesses whose reviews contains this word. We will also record the review iD, and the position of the word in that review. The dictionary is dumped as json to the inverted index file.

To save space, line number in the file is used instead of review_id and business_id

Why this structure?

There are two reasons for us to choose this structure:

First, space efficient. There is no redundant business or review ids in this structure. All data is contained, and can hardly have less content. Using the line number instead of the actual IDs also helps save storage space.

Second, query friendly. This is almost the same data structure we want to use for queries. By having this structure, we only need to load the map into memory and do not need to process the data for queries. The data structure for queries will be discussed later.

Storage Space

The index file turns out to be 192 MB(192737233 Bytes). 63% of the original review data file.

Query Program

Data Structure

The structure of the query program:
{word -> {business_id -> {review_id -> [positions]}}} in json form

The data structure of the query program is the same as the data structure in the inverted index file. We just load the json flie into memory directly, and we get our data. Again, the dictionary maps each word to another dictionary, which maps business ids to review dictionary. The review dictionay maps review to a list of positions that the word appears in the review.

Since we use line number instead as ids in the inverted index file, we also need to load the the yelp business data json file to convert business line number back to its business information. To do this, we create another dictionary that maps line number to the business information from the yelp business file.

Processing Review Texts

When processing the review texts, we did the following tokenizing steps:

  1. Lowercase all caharacters
  2. Apply stemming using Porter Stemmer
  3. Strip punctuations
  4. Replace two or more occurances of the same character with two occurrences.
  5. ignore words that don't start with an alphabet
  6. Use the strop words list

Making Queries

There are three kind of queries the user can make: single word query, free text query, and phrase query.

When the query program reads in the user input, it first parse by line. Then it checks if the query starts and ends with a quotation mark. If so, then process it as a phrase query. Otherwise, the program splits the input. If there is only one word then it's processed as single word query, otherwise free text query.

Ranking Algorithm

We use term frequency - inverse document frequency (tf-idf) as the primary ranking machanism. It takes into account the number of times a term appears in the specific business, and the number of businesses that contain this term.

In addition to tf-idf, we also take into stars, review count into consideration when computing the relavence between and a business. We use the heuristic function relevancy(query, business) = sum(tf_idf) * log(review_count) * stars to compute the final score. This function also puts a large weight on review count, since the more review count a businesses has, the better it servers its user.

The herustic function did a good job up-ranking the business that has higher review counts. The ones without herustic may look more relavant to the search term on its title or review texts, but on average the top ones using heuristic will receive better user feedback.
